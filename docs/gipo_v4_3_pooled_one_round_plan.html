GIPO V4.3: Pooled One-Round Calibration

Canonical implementation plan for one consistent optimization distribution, one teacher, one student, and no final validation selection.

0. Final decision

Use one pooled calibration run. Treat 20% of train + validation as one optimization set, sampled proportionally from both splits. Run one BO phase with the configured candidate budget. Train one ranking-first teacher and one SER-initialized student. Select the final student by teacher utility under a hard geometry guard, not by lowest internal loss.

1. Design constraints

ObservationImplementation implication for V4.3
Validation-only policy selection is not part of the canonical workflow. No separate validation policy selection is needed. Use sampled validation examples as calibration data, not as a selector.
The unguarded steps25 teacher-top looked attractive internally but was geometry-riskier and worse. The geometry guard is a hard feasibility constraint and must be applied after seed-mean projection.
BO-only teacher rows, uniform student initialization, and lowest-internal-loss final selection are not canonical. Keep fixed schedules as teacher anchors, initialize the student from SER-PTG, and select by guarded teacher utility.

3. One BO phase

Use one pooled BO buffer. Evaluate the configured candidate-family budget against the single calibration pool.

Do not restart BO around the former validation set. There is no Train20 stage and ValCal stage in V4.3. All candidates live in one buffer and receive one pooled reward.

4. Reward

Evaluate all six fixed schedules on C. Use them both as reward references and teacher anchor rows.

UC(ρ; s,b) = − 1/2 [ log(CRPSC(ρ;s,b) / CRPSC,best-fixed(s,b)) + log(MASEC(ρ;s,b) / MASEC,best-fixed(s,b)) ].
  • One scalar composite reward only.
  • No CRPS-only or MASE-only policy yet.
  • No split-aware train/val penalty in the main run.
  • Report CRPS and MASE separately after locked test.

8. Locked test

  1. Freeze one schedule JSON before locked test.
  2. Run locked test once.
  3. Compare against the six fixed schedules.
  4. Report CRPS, MASE, wins vs uniform, gap to per-cell best fixed, geometry diagnostics, and latency.
Do not choose among implementation variants by locked-test result. V4.3 must be frozen before the locked-test run.

9. Minimal pseudocode

# V4.3 pooled one-round calibration

C = make_calib_pool(train_split, validation_split, fraction=0.20)
fixed_rows = eval_fixed_schedules(C)      # also teacher anchors
ser_row    = eval_ser_ptg(C)              # anchor and initialization reference

candidates = build_one_phase_candidates(
    base="SER-PTG",
    basis="7D_lowrank",
    total_budget=configured_candidate_budget,
    mix=["ser_zero", "hand_bumps", "sobol", "one_logei_batch"]
)
bo_rows = eval_candidates(C, candidates)

all_rows = compute_paired_best_fixed_reward(
    rows=bo_rows + fixed_rows + [ser_row],
    fixed_refs=fixed_rows,
    metric="composite_CRPS_MASE"
)

teacher = train_one_teacher(
    rows=all_rows,
    loss="rank + 0.25*huber + fixed_crossing",
    select_by="heldout_pairwise_spearman_topk_crossing"
)

student_candidates = train_one_student(
    init="SER-PTG",
    teacher=teacher,
    seeds=[0,1,2],
    modes=[5,10,15,20,25]
)

selected = select_by_teacher_utility_with_geometry_guard(student_candidates)
final_schedule = seed_mean_then_guard(selected)
export(final_schedule)
locked_test_once(final_schedule)

10. Do not change now

  • Do not add metric-specific policies.
  • Do not use two teachers or teacher ensembles.
  • Do not run a separate former-validation BO stage.
  • Do not initialize the student from uniform.
  • Do not use lowest internal loss as the final schedule selector.
  • Do not relax the geometry guard.
  • Do not deploy the direct BO oracle as the main method.
